## 컨텍스트

- 현재 git 상태: !`git status --short`
- 변경 내역 (unstaged + staged): !`git diff HEAD`
- 최근 커밋 로그: !`git log --oneline -5`

## 작업

1. **변경사항 확인**
   - `git status`로 모든 변경 파일 목록 표시
   - `git diff`로 구체적인 변경 내용 표시
   - 변경사항이 없으면 "커밋할 변경사항이 없습니다"라고 안내하고 종료

2. **커밋 메시지 작성**
   - `$ARGUMENTS`가 비어있으면 (사용자가 인자를 주지 않으면):
     - 변경 내용을 분석하여 적절한 커밋 메시지를 제안한다
     - 제안 형식: `[타입]: 설명` (예: `feat: 새로운 기능 추가`, `fix: 버그 수정`)
     - 타입: `feat`, `fix`, `docs`, `style`, `refactor`, `chore`, `test` 중 선택
   - `$ARGUMENTS`가 있으면:
     - 사용자가 입력한 커밋 메시지를 그대로 사용

3. **커밋 메시지 확인**
   - 제안/입력된 커밋 메시지를 사용자에게 보여줌
   - 사용자 확인: "이 메시지로 커밋하시겠습니까? (yes/no)"
   - 사용자가 거부하면:
     - "커밋이 취소되었습니다"라고 안내하고 종료
     - (선택사항) 다른 메시지 입력 기회 제공
   - 사용자가 수락하면: 다음 단계로 진행

4. **커밋 생성 및 푸시**
   - 변경사항을 git에 추가: `git add .`
   - 커밋 생성: `git commit -m "[메시지]"`
   - 커밋 성공 시:
     - 최신 커밋 정보 표시: `git log --oneline -1`
     - "커밋이 성공적으로 생성되었습니다" 메시지 표시

### 주의사항

- **`.env` 또는 민감한 파일이 포함된 경우**: 경고를 표시하고 사용자 확인 필수
- **빈 커밋 (Empty commit)**: 변경사항이 없으면 커밋하지 않음
- **사전 커밋 훅 (pre-commit hook)**: 훅 실패 시 에러 메시지 표시하고 커밋하지 않음
